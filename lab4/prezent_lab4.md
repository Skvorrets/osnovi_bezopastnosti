# Актуальность темы
- В Linux разграничение доступа реализуется не только базовыми правами `rwx`, но и расширенными атрибутами.
- Атрибуты `a` и `i` позволяют дополнительно ограничить операции над файлами.
- Практическое понимание этих механизмов важно для администрирования и защиты данных.

# Цель работы
Получить практические навыки работы в консоли Linux с расширенными атрибутами файлов и исследовать влияние атрибутов `a` и `i` на изменение файлов.

# Исходные команды
```bash
lsattr /home/guest/dir1/file1
chmod 600 /home/guest/dir1/file1
chattr +a /home/guest/dir1/file1
```
- `lsattr` показывает расширенные атрибуты.
- `chmod 600` даёт доступ только владельцу.
- `chattr +a` пытается включить режим дозаписи.

---

# Атрибут `a` (append only)
- Устанавливается только суперпользователем.
- Разрешает добавление данных в конец файла.
- Запрещает обычную перезапись, удаление и переименование.

---

# Результат работы с атрибутом `a`
- Пользователь `guest` не может сам установить `a`.
- После установки от `root` файл допускает только дозапись.
- Команды перезаписи завершаются отказом.

---

# Атрибут `i` (immutable)
- Делает файл неизменяемым.
- Запрещает дозапись, перезапись, удаление и переименование.
- Снимается только суперпользователем.

---

# Сравнение `a` и `i`
- `a` — разрешена только дозапись.
- `i` — любые изменения запрещены.
- Оба атрибута усиливают защиту файла по сравнению с обычными правами доступа.
