## 2. Ход выполнения работы

### Шаг 1. Создание пользователя guest и первичная проверка окружения

На первом этапе была создана учётная запись пользователя `guest`, после чего для неё был задан пароль. Затем был выполнен вход в систему от имени пользователя `guest` и определена текущая рабочая директория. С помощью команды `whoami` было подтверждено имя текущего пользователя.

**Команды:**
```bash
useradd guest
passwd guest
pwd
whoami
```

**Наблюдение.** После входа в систему пользователь `guest` оказался в своей домашней директории `/home/guest`, что совпадает с приглашением командной строки.


### Шаг 2. Определение идентификаторов пользователя и просмотр /etc/passwd

Далее были исследованы идентификаторы пользователя и его группа. Для этого использовались команды `id`, `groups` и фильтрация файла `/etc/passwd` по имени `guest`.

**Команды:**
```bash
id
groups
cat /etc/passwd | grep guest
```

**Наблюдение.** Вывод команд подтвердил соответствие `uid` и `gid` пользователя `guest`, а запись в `/etc/passwd` показала домашнюю директорию и оболочку входа.


### Шаг 3. Просмотр директории /home, атрибутов и создание dir1

После этого был исследован каталог `/home`, а также его расширенные атрибуты. В домашней директории пользователя была создана поддиректория `dir1`, после чего просмотрены её права доступа и атрибуты.

**Команды:**
```bash
ls -l /home/
lsattr /home
mkdir dir1
ls -ld dir1
lsattr dir1
```

**Наблюдение.** Новая директория `dir1` получила стандартные права доступа для владельца, группы и остальных пользователей. Специальных расширенных атрибутов на директории установлено не было.


### Шаг 4. Снятие всех прав с dir1 и попытка создания файла

На следующем этапе у директории `dir1` были сняты все права. Затем была предпринята попытка создать внутри неё файл `file1`.

**Команды:**
```bash
chmod 000 dir1
ls -ld dir1
echo "test" > /home/guest/dir1/file1
ls -l /home/guest/dir1
```

**Наблюдение.** Операция создания файла завершилась отказом в доступе, так как у владельца отсутствовали права на выполнение операций внутри директории. Файл создан не был.


### Шаг 5. Исследование режима d--x------ для директории

Затем были проверены действия при наличии только бита выполнения у директории. Пользователь смог перейти в каталог, однако просматривать список файлов в нём уже не мог.

**Команды:**
```bash
chmod 100 dir1
chmod 000 dir1/file1
ls -ld dir1
cd dir1
pwd
ls
```

**Наблюдение.** Наличие только права `x` на директорию позволяет перейти в неё, но не даёт права на просмотр содержимого каталога.


### Шаг 6. Проверка полного доступа владельца

После этого для директории и файла были установлены полные права владельца. Были проверены операции создания, записи, чтения, переименования и удаления.

**Команды:**
```bash
chmod 700 dir1
chmod 700 dir1/file1
touch dir1/file1
echo test > dir1/file1
cat dir1/file1
mv dir1/file1 dir1/file1.bak
ls dir1
```

**Наблюдение.** При правах `700` на директорию и файл все основные операции доступны владельцу: создание, запись, чтение, просмотр содержимого, смена директории и переименование.


## 3. Таблица 2.1. Установленные права и разрешённые действия

| Права директории | Права файла | Создание файла | Удаление файла | Запись в файл | Чтение файла | Смена директории | Просмотр файлов в директории | Переименование файла | Смена атрибутов файла |
|---|---|---:|---:|---:|---:|---:|---:|---:|---:|
| d (000) | (000) | - | - | - | - | - | - | - | - |
| d--x------ (100) | (000) | - | - | - | - | + | - | - | + |
| drwx------ (700) | -rwx------ (700) | + | + | + | + | + | + | + | + |

## 4. Таблица 2.2. Минимальные права для совершения операций

| Операция | Минимальные права на директорию | Минимальные права на файл |
|---|---|---|
| Создание файла | `wx` | не требуются |
| Удаление файла | `wx` | не требуются |
| Чтение файла | `x` | `r` |
| Запись в файл | `x` | `w` |
| Переименование файла | `wx` | не требуются |
| Создание поддиректории | `wx` | не требуются |
| Удаление поддиректории | `wx` | не требуются |
