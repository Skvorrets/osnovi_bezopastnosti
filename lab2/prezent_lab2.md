# Создание пользователя и проверка окружения

```bash
useradd guest
passwd guest
pwd
whoami
id
groups
```

- Пользователь `guest` находится в домашнем каталоге `/home/guest`.
- Значения `uid` и `gid` совпадают с записью в `/etc/passwd`.

# Работа с dir1

```bash
mkdir dir1
chmod 000 dir1
echo "test" > /home/guest/dir1/file1
chmod 100 dir1
cd dir1
ls
chmod 700 dir1
```

- При `000` операции внутри каталога запрещены.
- При `100` можно войти в каталог, но нельзя просматривать содержимое.
- При `700` владельцу доступны все операции.

# Таблица 2.2

| Операция | Права на директорию | Права на файл |
|---|---|---|
| Создание файла | `wx` | не требуются |
| Удаление файла | `wx` | не требуются |
| Чтение файла | `x` | `r` |
| Запись в файл | `x` | `w` |
| Переименование файла | `wx` | не требуются |
| Создание поддиректории | `wx` | не требуются |
| Удаление поддиректории | `wx` | не требуются |

---

# Выводы

- Права на директорию и файл влияют на разные операции.
- Право `x` на директорию отвечает за вход в каталог.
- Право `r` на директорию отвечает за просмотр содержимого.
- Сочетание `w` и `x` необходимо для создания, удаления и переименования объектов.
