# Цель работы

Получить практические навыки работы в Linux с правами доступа для групп пользователей и определить минимальные права, необходимые для выполнения типовых операций в общей директории.

# Создание пользователей и группы

```bash
useradd guest
passwd guest
useradd guest2
passwd guest2
gpasswd -a guest2 guest
```

# Проверка пользователей и групп

```bash
pwd
whoami
groups guest
groups guest2
id -Gn
id -G
cat /etc/group | grep guest
newgrp guest
```

![](lab3_pack/imgs/s3.png)

# Изменение прав каталога

```bash
chmod g+rwx /home/guest
chmod 000 dir1
```

- при правах `000` пользователь `guest2` не может войти в каталог
- права директории влияют на доступ к объектам внутри неё

# Таблица 3.2

| Операция | Минимальные права на директорию | Минимальные права на файл |
|---|---|---|
| Создание файла | wx | — |
| Удаление файла | wx | — |
| Чтение файла | x | r |
| Запись в файл | x | w |
| Переименование файла | wx | — |
| Создание поддиректории | wx | — |
| Удаление поддиректории | wx | — |

---

# Выводы

- доступ определяется одновременно правами файла и правами каталога
- для входа в каталог нужен `x`
- для просмотра содержимого каталога нужен `r`
- для создания, удаления и переименования файлов нужны `w` и `x`
- включение `guest2` в группу `guest` позволяет выполнять операции над групповыми объектами
